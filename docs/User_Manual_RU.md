## Введение
**VeriPharm** (внутреннее название проекта *MedCompatibility*) — это кроссплатформенное мобильное и десктопное приложение, разработанное на базе .NET MAUI. Приложение предназначено для проверки совместимости лекарственных препаратов, ведения истории рецептов, а также обеспечения взаимодействия между врачами и пациентами.

---

## Подробное руководство пользователя
Приложение поддерживает ролевую модель доступа. После регистрации и авторизации интерфейс адаптируется под назначенную вам роль.

### Для пациентов (Patient)
- **Главная страница (Home)**: Быстрый доступ к сканированию штрихкодов препаратов, просмотру текущих рецептов и расписания приема лекарств.
- **Сканирование (Scan)**: Возможность отсканировать штрихкод лекарства через камеру устройства для быстрого поиска его в базе данных.
- **Проверка совместимости (Compatibility)**: Позволяет добавить несколько препаратов и запустить кросс-анализ на предмет побочных эффектов и негативных взаимодействий (включая AI-анализ).
- **Расписание и история (Schedule & History)**: Просмотр графика приема выписанных лекарств и истории проверок.
- **Профиль (Profile)**: Настройки аккаунта, персональные данные и возможность экспорта отчетов (в PDF).

### Для врачей (Doctor)
- **Главная страница врача (Doctor Home)**: Сводная статистика по пациентам и активным рецептам.
- **Список пациентов (Patients)**: Просмотр прикрепленных пациентов.
- **Карточка пациента (Patient Card)**: Просмотр истории болезни пациента, его текущих медикаментов и создание новых рецептов (Prescriptions).
- **Кросс-анализ (Cross Analysis)**: Расширенный функционал проверки совместимости назначаемых препаратов с текущим план лечения пациента.

### Для администраторов (Admin)
- **Панель управления (Admin Home)**: Управление базой данных системы.
- **Управление пользователями (Users)**: Назначение ролей (Врач, Пациент, Админ), блокировка аккаунтов.
- **Справочник лекарств (Medicines)**: Добавление, редактирование и удаление лекарственных препаратов, производителей и активных веществ.
- **Взаимодействия (Interactions)**: Управление справочником совместимости препаратов (риски, типы взаимодействий).
- **Системные логи (System Logs)**: Просмотр журнала системных событий и ошибок приложения.

## Инструкции по развертыванию
Приложение является клиентом .NET MAUI с прямым подключением к базе данных MySQL.

### Требования к окружению
- **СУБД**: MySQL Server (версия 8.0 и выше).
- **SDK**: .NET 8.0 SDK (или новее) с установленной рабочей нагрузкой MAUI (`dotnet workload install maui`).
- **IDE**: Visual Studio 2022 / JetBrains Rider / VS Code (с расширениями C# и MAUI).

### Шаги по запуску

1. **Клонирование репозитория:**
   Откройте папку проекта `MedCompatibility`.

2. **Настройка конфигурации:**
   Создайте файл `appsettings.json` в корневой директории проекта `MedCompatibility` (рядом с `appsettings.Example.json`).
   ```json
   {
     "Database": {
       "Local": {
         "Host": "127.0.0.1",
         "Port": 3306,
         "Name": "med_compat_db",
         "User": "root",
         "Password": "your_password"
       }
     },
     "AI": {
       "ApiKey": "ваш-ai-api-ключ"
     }
   }
   ```
   *Примечание: Приложение использует конфигурацию `Database` для создания строки подключения.*

3. **База данных:**
   Приложение использует Entity Framework Core (`DrugContext`). Для инициализации БД примените миграции:
   ```bash
   dotnet ef database update
   ```

4. **Сборка и запуск:**
   Запустите приложение на нужной платформе:
   - **Windows**: Выберите `Windows Machine` в Visual Studio или выполните `dotnet build -t:Run -f net8.0-windows10.0.19041.0`.
   - **Android**: Подключите эмулятор или устройство и выберите профиль Android.

## Описание архитектуры
VeriPharm построен по паттерну **MVVM** (Model-View-ViewModel), что обеспечивает четкое разделение бизнес-логики и пользовательского интерфейса.

### Основные компоненты
- **Views (Pages/Popups)**: XAML файлы, отвечающие за UI. Используется библиотека `UraniumUI` для реализации Material Design и `CommunityToolkit.Maui` для всплывающих окон и утилит.
- **ViewModels**: Классы, содержащие логику представления. Они связываются с Views через Data Binding.
- **Models**: Сущности предметной области (`User`, `Medicine`, `Prescription`, `Interaction`, `SystemLog` и т.д.), которые маппятся в таблицы MySQL через EF Core.
- **Services (Business Logic)**: Набор Dependency Injection (DI) сервисов, инкапсулирующих работу с базой данных, внешними API (AI) и устройством.
- **Data Access Layer**: `DrugContext` — контекст Entity Framework Core, напрямую подключающийся к базе данных (через провайдер `Pomelo.EntityFrameworkCore.MySql`).

### Особенности
- **Локальные уведомления**: Реализованы через `Plugin.LocalNotification` (напоминания о приеме лекарств).
- **Сканер штрихкодов**: Интегрирован через `ZXing.Net.Maui`.
- **Создание отчетов**: Генерация PDF с назначениями и историей через `IPdfReportService`.

## Описание API и Сервисов
Поскольку приложение обращается к БД напрямую, роль классического REST API выполняют внутренние сервисы (зарегистрированные в `MauiProgram.cs`).

### Основные сервисы предметной области (Domain Services)
- `IAuthService` — отвечает за регистрацию, авторизацию и хэширование паролей пользователей.
- `IUserService` — управление профилями, смена ролей, получение списков пациентов для врачей.
- `IMedicineService` — CRUD-операции над справочником лекарств, активных веществ и производителей.
- `IInteractionService` — поиск конфликтов между лекарствами. Проверяет связи в таблицах взаимодействий на основе активных веществ.
- `IPrescriptionService` — назначение лекарств пациенту, создание графиков приема.
- `IScanService` — обработка результатов сканирования штрихкодов.

### Интеграционные сервисы
- **IAiInteractionService / IAiHealthService**
  Отправляют запросы к внешним Large Language Models (LLM) для интеллектуального анализа рецептов, поиска неочевидных побочных эффектов и советов по здоровью. Используется ключ из конфигурации `AI:ApiKey`.
- **IPdfReportService**
  Формирует PDF-документы на стороне клиента для печати рецептов или выгрузки истории болезни.
- **INotificationService**
  Регистрация отложенных пуш-уведомлений в планировщике ОС (для напоминания принять таблетку).

### Внешнее взаимодействие
Хотя у приложения нет собственного бэкенда (кроме MySQL), оно выполняет исходящие HTTPS запросы к:
1. Провайдеру ИИ (OpenAI / Anthropic / др. в зависимости от реализации `AiInteractionService`).
2. Облачной СУБД (если настроено подключение к Cloud MySQL, как указано в конфигурации).
